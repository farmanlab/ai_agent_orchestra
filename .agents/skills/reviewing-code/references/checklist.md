# Code Review Checklist

## Architecture & Design
- [ ] レイヤー間の依存関係が適切
- [ ] 単一責務の原則に従っている
- [ ] 開放閉鎖の原則に従っている
- [ ] 適切な抽象化レベル
- [ ] デザインパターンの適切な使用

## Code Quality
- [ ] 関数は30行以内
- [ ] ネストは3段階以内
- [ ] 命名は意図を明確に表現
- [ ] マジックナンバーがない
- [ ] コメントは「なぜ」を説明

## Error Handling
- [ ] エラーは適切に捕捉
- [ ] エラーメッセージは明確
- [ ] リソースの解放を保証
- [ ] 異常系のテストあり

## Performance
- [ ] 不要な計算を避けている
- [ ] 適切なデータ構造を使用
- [ ] N+1問題がない
- [ ] キャッシュを適切に使用

## Security
- [ ] 入力は検証済み
- [ ] SQLインジェクション対策
- [ ] XSS対策
- [ ] 認証・認可の実装が適切
- [ ] 機密情報がログに出力されない

## Testing
- [ ] 新規コードにテストあり
- [ ] エッジケースをカバー
- [ ] 異常系をカバー
- [ ] テストは独立して実行可能
- [ ] テストは高速（単体テスト1秒以内）

## Documentation
- [ ] APIドキュメント更新
- [ ] READMEの更新（必要に応じて）
- [ ] 複雑なロジックにコメント
- [ ] 破壊的変更の記録

## Git
- [ ] コミットメッセージは明確
- [ ] コミットは論理的な単位
- [ ] 不要なファイルが含まれていない
- [ ] コンフリクトが解決済み
